# Assistant Run Instructions (English)

Purpose
- Provide step-by-step commands for the assistant to run the FleetManagement.Equipment microservice locally.

Prerequisites
- .NET SDK installed and available on PATH (`dotnet --version`)
- Docker or Podman available to run SQL Server container
- (Optional) EF Core tools: `dotnet-ef` for creating/applying migrations
- Run all commands from the repository root (the project folder where the solution and subprojects live). Do NOT hardcode absolute home paths.

1) Start local SQL Server (choose one: Docker or Podman)

Using Docker (Linux/macOS/Windows)

```sh
# Example (local development, example SA password; do NOT use in production)
# Run this from the repository root (or any working folder) — no absolute path required.

docker run -e "ACCEPT_EULA=Y" -e "MSSQL_SA_PASSWORD=MyPassword123!!" -e "MSSQL_PID=StandardDeveloper" -p 1433:1433 --name sql2025 --hostname sql2025 -d mcr.microsoft.com/mssql/server:2025-latest

# Verify container is running:
docker ps --filter "name=sql2025"

# Stop and remove when done:
docker stop sql2025 && docker rm sql2025
```

Using Podman (Linux/macOS)

```sh
podman run -e "ACCEPT_EULA=Y" -e "MSSQL_SA_PASSWORD=MyPassword123!!" -e "MSSQL_PID=StandardDeveloper" -p 1433:1433 --name sql2025 --hostname sql2025 -d mcr.microsoft.com/mssql/server:2025-latest

# Verify:
podman ps --filter "name=sql2025"

# Stop and remove when done:
podman stop sql2025 && podman rm sql2025
```

2) Configure connection string
- Edit `FleetManagement.Equipment.API/appsettings.Development.json` and set `ConnectionStrings:SqlDatabase` to (example):

```json
"Server=localhost,1433;Database=FleetManagement.Equipment;User Id=sa;Password=MyPassword123!!;Encrypt=True;TrustServerCertificate=True;"
```

3) Restore dependencies

```sh
# From repository root (no hardcoded home path):
dotnet restore
```

4) (Optional) Create and apply EF Core migrations

```sh
# Install EF Core CLI if missing
dotnet tool install --global dotnet-ef

# Create migration (adds to Infrastructure project, starts from API project)
dotnet ef migrations add InitialCreate -p ./FleetManagement.Equipment.Infrastructure/FleetManagement.Equipment.Infrastructure.csproj -s ./FleetManagement.Equipment.API/FleetManagement.Equipment.API.csproj

# Apply migrations to database
dotnet ef database update -p ./FleetManagement.Equipment.Infrastructure/FleetManagement.Equipment.Infrastructure.csproj -s ./FleetManagement.Equipment.API/FleetManagement.Equipment.API.csproj
```

5) Run the API

```sh
# From repository root (project-relative path):
dotnet run --project FleetManagement.Equipment.API/FleetManagement.Equipment.API.csproj
```

6) Build and test (optional)

```sh
dotnet build -c Debug
dotnet test
```

Windows-specific notes
- On Windows use the repository folder path (e.g., `C:\path\to\repo`) and run the same `dotnet` commands from that folder.
- Docker Desktop on Windows supports the same `docker run` command. In PowerShell you may prefer `--env KEY=VALUE` or avoid shell-quoting issues.
- Example (PowerShell-friendly):

```powershell
docker run --env ACCEPT_EULA=Y --env MSSQL_SA_PASSWORD=MyPassword123!! --env MSSQL_PID=StandardDeveloper -p 1433:1433 --name sql2025 --hostname sql2025 -d mcr.microsoft.com/mssql/server:2025-latest
```
